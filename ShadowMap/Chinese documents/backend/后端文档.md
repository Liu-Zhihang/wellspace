# 后端文档（配置与自检）

本文档聚焦「如何把后端与 GeoServer/WFS 跑通」以及常见自检命令。更完整的英文说明见：`ShadowMap/shadow-map-backend/README.md`。

## 1) 快速启动（开发）

```bash
cd ShadowMap/shadow-map-backend
cp .env.example .env
npm install
npm run dev
```

默认端口通常为 `3001`（以 `.env` 为准）。健康检查：

```bash
curl -s http://localhost:3001/api/health | head -c 200 && echo
```

## 2) GeoServer / WFS 对接

后端通过 `.env` 配置 WFS 代理（示例变量名以 `ShadowMap/shadow-map-backend/.env.example` 为准）：

- `BUILDING_WFS_BASE_URL`：例如 `http://localhost:8080/geoserver/shadowmap/wfs`
- `BUILDING_WFS_TYPE_NAME`：例如 `shadowmap:buildings`
- `BUILDING_WFS_TILE_CATALOG_PATH`：瓦片范围 catalog（可选但强烈推荐）

连通性测试：

```bash
curl -s http://localhost:8080/geoserver/wfs?service=WFS&version=1.1.0&request=GetCapabilities | head -c 200 && echo
curl -s http://localhost:3001/api/wfs-buildings/test | head -c 300 && echo
```

## 3) 前端对接

前端通过 `VITE_BACKEND_BASE_URL` 指向后端，例如：

```bash
export VITE_BACKEND_BASE_URL="http://localhost:3001"
```

然后在 `ShadowMap/shadow-map-frontend/react-shadow-app/` 启动 Vite 开发服务器。

## 4) 与批量计算（Python）关系说明

- 纯 Python 批量计算（mobility sunlight/shadow）不依赖后端 HTTP 服务，主要通过 `ShadowMap/.shadowmap.env` 配置本地数据路径。
- GeoServer/WFS 主要服务于 Demo 与交互式前端建筑查询。
