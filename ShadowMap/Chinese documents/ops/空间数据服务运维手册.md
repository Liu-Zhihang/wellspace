# 空间数据服务运维手册（PostGIS + GeoServer + ShadowMap）

最后更新：2025-12-19

本手册将原先分散的两份文档（部署/导入）合并为一份“可执行主线”，用于在工作站上维护：

- PostGIS（建筑矢量数据）
- GeoServer（WFS/WMS 服务）
- ShadowMap 后端（WFS 代理与自检接口）

> 注意：文档中不应出现真实密码/Token；仅使用变量名或占位符。

---

## 0. 快速自检（出现问题先跑这里）

```bash
# GeoServer 是否在线（WFS capabilities）
curl -s "http://localhost:8080/geoserver/wfs?service=WFS&version=1.1.0&request=GetCapabilities" | head -c 200 && echo

# PostGIS 是否可连（示例；按你的库名调整）
PGPASSWORD="<GISUSER_PASSWORD>" psql -h localhost -U gisuser -d shadowmap_gis -c "select 1;"

# ShadowMap backend 是否在线
curl -s http://localhost:3001/api/health | head -c 200 && echo
curl -s http://localhost:3001/api/wfs-buildings/test | head -c 300 && echo
```

## 1. 架构与端口

典型部署（同机）：

- GeoServer：`http://<HOST>:8080/geoserver`
- PostGIS：`<HOST>:5432`
- ShadowMap backend：`http://<HOST>:3001`

前端（Windows/其他机器）通过局域网访问 backend，再由 backend 代理 WFS/DEM 等请求。

## 2. 安装与部署（一次性）

### 2.1 PostgreSQL + PostGIS

```bash
sudo apt update
sudo apt install -y postgresql postgresql-contrib postgis postgresql-postgis
sudo systemctl enable --now postgresql
```

创建数据库与用户（示例）：

```bash
sudo -u postgres psql <<'SQL'
CREATE DATABASE shadowmap_gis;
CREATE USER gisuser WITH PASSWORD '<GISUSER_PASSWORD>';
GRANT ALL PRIVILEGES ON DATABASE shadowmap_gis TO gisuser;
\c shadowmap_gis
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS postgis_topology;
GRANT ALL ON ALL TABLES IN SCHEMA public TO gisuser;
GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO gisuser;
SQL
```

远程访问（按需；仅局域网）：

- `postgresql.conf`：`listen_addresses='*'`
- `pg_hba.conf`：只放行你的网段（例如 `192.168.0.0/16 md5`）

修改后：

```bash
sudo systemctl restart postgresql
```

### 2.2 GeoServer（systemd）

安装 Java（示例使用 OpenJDK 11）：

```bash
sudo apt install -y openjdk-11-jdk
java -version
```

GeoServer 安装与 systemd 配置请参考官方文档或你实验室的标准安装脚本；关键要求：

- `GEOSERVER_DATA_DIR` 固定且可读写（例如 `/opt/geoserver/data_dir`）
- GeoServer 进程用户（例如 `geoserver`）对 data_dir 有权限

常用操作：

```bash
sudo systemctl restart geoserver
sudo systemctl status geoserver --no-pager
```

## 3. 建筑数据入库（GeoJSON → PostGIS buildings）

### 3.1 约定：`buildings` 表与关键字段

建议 `buildings` 表至少包含：

- `geom`：几何（多边形；建议 EPSG:4326）
- `height`：高度（可为空；下游有默认值兜底）
- `tile_id`：瓦片 ID（强烈建议，用于后端限流与加速）

索引建议：

```sql
CREATE INDEX IF NOT EXISTS buildings_tile_idx ON buildings(tile_id);
CREATE INDEX IF NOT EXISTS buildings_geom_idx ON buildings USING GIST (geom);
VACUUM ANALYZE buildings;
```

### 3.2 单瓦片导入（推荐方式）

1) 探查 GeoJSON 图层名：

```bash
ogrinfo -ro -so ~/DATASET/<tile_id>.geojson
```

2) 导入到 `buildings`（追加）并写入 `tile_id`：

```bash
export PGPASSWORD="<GISUSER_PASSWORD>"
ogr2ogr -f "PostgreSQL" \
  PG:"host=localhost dbname=shadowmap_gis user=gisuser" \
  ~/DATASET/<tile_id>.geojson \
  -nln buildings \
  -append \
  -nlt PROMOTE_TO_MULTI \
  -dialect SQLite \
  -sql "SELECT *, '<tile_id>' AS tile_id FROM \"<图层名>\"" \
  -progress
unset PGPASSWORD
```

3) 校验：

```bash
PGPASSWORD="<GISUSER_PASSWORD>" psql -h localhost -U gisuser -d shadowmap_gis -c \
  "select tile_id, count(*) from buildings group by tile_id order by tile_id;"
```

### 3.3 批量导入目录（northamerica 等）

当文件名就是瓦片 ID（如 `w030_n20_w025_n15.geojson`）：

```bash
DIR=~/DATASET/LoD1/northamerica
export PGPASSWORD="<GISUSER_PASSWORD>"

for f in "$DIR"/*.geojson; do
  tile_id="$(basename "$f" .geojson)"
  echo "[Import] $tile_id"

  ogr2ogr -f "PostgreSQL" \
    PG:"host=localhost dbname=shadowmap_gis user=gisuser" \
    "$f" \
    -nln buildings \
    -append \
    -nlt PROMOTE_TO_MULTI \
    -dialect SQLite \
    -sql "SELECT *, '${tile_id}' AS tile_id FROM \"${tile_id}\"" \
    -progress
done

unset PGPASSWORD
```

> 若遇到“图层不存在”：先对单个文件 `ogrinfo -ro -so`，把 SQL 里的 `"${tile_id}"` 改为真实图层名。

## 4. DEM 部署（GeoTIFF → GeoServer ImageMosaic）

约定：

- DEM 放在 GeoServer data_dir 内的目录，例如：
  - `/opt/geoserver/data_dir/data/rasters/dem_global/`
- GeoServer 中使用 ImageMosaic store（例如 `shadowmap:dem_global`）

部署步骤：

```bash
sudo mkdir -p /opt/geoserver/data_dir/data/rasters/dem_global
sudo chown -R geoserver:geoserver /opt/geoserver/data_dir/data/rasters

sudo find ~/DATASET/<tile_id> -type f -name "*.tif" \
  -exec mv {} /opt/geoserver/data_dir/data/rasters/dem_global/ \;

sudo chown geoserver:geoserver /opt/geoserver/data_dir/data/rasters/dem_global/*.tif
```

刷新 GeoServer（任选其一）：

- Web UI：进入 store，点击 `Save` 触发重索引
- REST：
  ```bash
  curl -u admin:<GEOSERVER_PASSWORD> -XPOST http://localhost:8080/geoserver/rest/reload
  ```

## 5. GeoServer 发布与验证

WFS（矢量）验证：

```bash
curl "http://localhost:8080/geoserver/wfs?service=WFS&version=1.0.0&request=GetFeature&typeName=shadowmap:buildings&cql_filter=tile_id='<tile_id>'&maxFeatures=5"
```

WMS（DEM）验证（示例 bbox 需替换）：

```bash
curl "http://localhost:8080/geoserver/wms?service=WMS&request=GetMap&layers=shadowmap:dem_global&bbox=<lon_min>,<lat_min>,<lon_max>,<lat_max>&width=512&height=512&srs=EPSG:4326&format=image/png" \
  --output <tile_id>_preview.png
```

## 6. ShadowMap 后端对接（WFS 代理）

后端通过 `ShadowMap/shadow-map-backend/.env` 配置：

- `BUILDING_WFS_BASE_URL`：例如 `http://localhost:8080/geoserver/shadowmap/wfs`
- `BUILDING_WFS_TYPE_NAME`：例如 `shadowmap:buildings`
- `BUILDING_WFS_TILE_CATALOG_PATH`：瓦片范围 catalog（可选但强烈推荐；用于 bbox→tile_id→WFS 过滤）

验证：

```bash
curl -s http://localhost:3001/api/wfs-buildings/test | head -c 300 && echo
```

## 7. 常见问题（排障）

### 7.1 PostGIS 连接失败（密码/权限）

现象：GeoServer 或 backend 日志出现 `FATAL: password authentication failed for user "gisuser"`。

排查：

```bash
PGPASSWORD="<GISUSER_PASSWORD>" psql -h localhost -U gisuser -d shadowmap_gis -c "select 1;"
```

修复方向：

- 确认 GeoServer store 的数据库连接字符串/密码已更新并保存
- 必要时重启 GeoServer：`sudo systemctl restart geoserver`

### 7.2 导入后几何为空（CRS 不匹配）

若源数据为 EPSG:3857 而目标 `geom` 在 EPSG:4326，可能导入为空几何。

建议先确认源 CRS（`ogrinfo`），再显式重投影：

```bash
ogr2ogr ... -s_srs EPSG:3857 -t_srs EPSG:4326 ...
```

### 7.3 GeoServer 字段缓存未更新

现象：WFS 里读不到新字段（如 `tile_id`）。

处理：对 store 点击 `Save` 或 `REST reload`，再重试请求。

### 7.4 tile catalog 不覆盖导致请求很慢/无数据

确保：

- PostGIS `buildings.tile_id` 已填充
- 后端 `buildingTiles.json`（或 `.env` 指定 catalog）覆盖导入范围
