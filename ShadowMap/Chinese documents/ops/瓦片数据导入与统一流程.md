# 瓦片数据导入与统一流程

## 概述

本流程用于在工作站上持续集成新的 5°×5° 瓦片数据（建筑矢量 + DEM 栅格），保持与现有 `shadowmap` 服务一致。按步骤执行即可完成上传、入库、发布以及验证。

## 前置准备

- PostgreSQL / PostGIS 中已存在 `buildings` 表，包含字段：`geom`、`height`、`var`、`source`、`region`（可选）、`tile_id` 等。
- GeoServer 服务运行中，`GEOSERVER_DATA_DIR` 为 `/opt/geoserver/data_dir`。
- DEM 统一放置在目录 `file:data/rasters/dem_global`（对应绝对路径 `/opt/geoserver/data_dir/data/rasters/dem_global`），使用 ImageMosaic Store `shadowmap:dem_global`。
- `gisuser` 账号具备对 `shadowmap_gis` 库的写权限；GeoServer 管理账号可通过 REST 或 Web UI 保存配置。

## 1. 文件整理

1. 将新瓦片的 GeoJSON 与 TIF 压缩包上传到 `~/DATASET`。命名建议统一为 `<tile_id>.geojson`、`<tile_id>.zip`。
2. 解压 DEM：
   ```bash
   cd ~/DATASET
   unzip <tile_id>.zip -d <tile_id>
   ```
3. 收集 DEM 文件路径：
   ```bash
   find ~/DATASET/<tile_id> -type f -name "*.tif"
   ```
   若数量巨大，可直接在第 3 步移动时使用 `find -exec`.

## 2. 建筑数据入库

1. 探查 GeoJSON 图层名称：
   ```bash
   ogrinfo -ro -so ~/DATASET/<tile_id>.geojson
   ```
2. 将瓦片写入 `buildings` 表，并附带唯一的 `tile_id`：
   ```bash
   export PGPASSWORD="<GISUSER_PASSWORD>"
   ogr2ogr -f "PostgreSQL" \
     PG:"host=localhost dbname=shadowmap_gis user=gisuser" \
     ~/DATASET/<tile_id>.geojson \
     -nln buildings \
     -append \
     -nlt PROMOTE_TO_MULTI \
     -dialect SQLite \
     -sql "SELECT *, '<tile_id>' AS tile_id FROM \"<图层名>\"" \
     -progress
   unset PGPASSWORD
   ```
3. 在 psql 中完成后置处理：
   ```sql
   SELECT tile_id, COUNT(*) FROM buildings GROUP BY tile_id ORDER BY tile_id;
   CREATE INDEX IF NOT EXISTS buildings_tile_idx ON buildings(tile_id);
   CREATE INDEX IF NOT EXISTS buildings_geom_idx ON buildings USING GIST (geom);
   VACUUM ANALYZE buildings;
   ```

### 2.1 批量导入一个目录（例如 northamerica）

如果你的 GeoJSON 文件名本身就是瓦片 ID（例如 `w030_n20_w025_n15.geojson`），建议直接用「不含扩展名」作为 `tile_id`，并用循环批量导入：

```bash
DIR=~/DATASET/LoD1/northamerica
export PGPASSWORD="<GISUSER_PASSWORD>"

for f in "$DIR"/*.geojson; do
  tile_id="$(basename "$f" .geojson)"
  echo "[Import] $tile_id"

  # GeoJSON 常见情况：图层名 = 文件名（不含扩展名）
  ogr2ogr -f "PostgreSQL" \
    PG:"host=localhost dbname=shadowmap_gis user=gisuser" \
    "$f" \
    -nln buildings \
    -append \
    -nlt PROMOTE_TO_MULTI \
    -dialect SQLite \
    -sql "SELECT *, '${tile_id}' AS tile_id FROM \"${tile_id}\"" \
    -progress
done

unset PGPASSWORD
```

> 若出现“图层不存在”，先对单个文件运行 `ogrinfo -ro -so <file>.geojson`，把 SQL 里的 `"${tile_id}"` 改成实际图层名。

## 3. DEM 文件部署

1. 创建（如未存在）DEM 目录并授予权限：
   ```bash
   sudo mkdir -p /opt/geoserver/data_dir/data/rasters/dem_global
   sudo chown -R geoserver:geoserver /opt/geoserver/data_dir/data/rasters
   ```
2. 将瓦片 TIF 移入目录：
   ```bash
   sudo find ~/DATASET/<tile_id> -type f -name "*.tif" \
     -exec mv {} /opt/geoserver/data_dir/data/rasters/dem_global/ \;
   sudo chown geoserver:geoserver /opt/geoserver/data_dir/data/rasters/dem_global/*.tif
   ```
3. 如果需要统一坐标系，可在移动前使用 `gdalwarp` 重投影；当前默认保留原始 `EPSG:32649`。

## 4. GeoServer 刷新

1. 登录 GeoServer Web 界面 → `Data → Stores`，点击 `shadowmap:dem_global` → 直接 `Save` 一次，触发 mosaic 重新索引。
   - 或使用 REST：
     ```bash
     curl -u admin:<GEOSERVER_PASSWORD> -XPOST http://localhost:8080/geoserver/rest/reload
     ```
2. 如需新增图层（首配置），在 `Data → Layers → Add a new layer` 中选择 `shadowmap:dem_global` Store，设置：
   - `Native SRS` 与 `Declared SRS`: `EPSG:32649`
   - `Compute from data` / `Compute from native bounds`
   - `Enabled` / `Advertised` 勾选 → `Save`
3. 若将来改用 `EPSG:4326`，只需将重投影后的目录另建 store，步骤相同。

## 5. 验证脚本

- WFS（矢量）：
  ```bash
  curl "http://localhost:8080/geoserver/wfs?service=WFS&version=1.0.0&request=GetFeature&typeName=shadowmap:buildings&cql_filter=tile_id='<tile_id>'&maxFeatures=5"
  ```
- WMS（DEM）：
  ```bash
  curl "http://localhost:8080/geoserver/wms?service=WMS&request=GetMap&layers=shadowmap:dem_global&bbox=<lon_min>,<lat_min>,<lon_max>,<lat_max>&width=512&height=512&srs=EPSG:4326&format=image/png" \
    --output <tile_id>_preview.png
  ```
  > 注：`bbox` 可使用瓦片对应的经纬度范围。

## 6. 日志与归档

- 记录导入信息（建议保存在 `~/DATASET/import-log.md`）：
  ```
  日期 | tile_id | 数据类型 | 源路径 | 行数/瓦片数 | 备注
  ```
- 保留原始 `.zip` 作为备份；GeoJSON 可同步到版本库（注意隐私）。
- 定期检查磁盘空间，必要时把历史瓦片迁移至大容量磁盘或对象存储。

## 7. 常见问题

- **GeoServer 返回 `Illegal property name: tile_id`**：刷新 PostGIS store（`Save` 或 REST reload），让字段缓存更新。
- **WMS 图像出不来**：确认 store 已发布、目录权限正确、`geoserver.log` 无 mosaic 错误。
- **`ogr2ogr` 提示图层不存在**：使用 `ogrinfo` 获取真实图层名，在 SQL 中引用 `"真实名称"`。
- **`tile_id` 漏写**：导入后用 `UPDATE buildings SET tile_id = '<tile_id>' WHERE tile_id IS NULL AND geom && ...` 补齐，再 `VACUUM ANALYZE`。
- **GeoJSON 几何全部导入为空**：若源数据坐标系为 EPSG:3857 而目标表 `geom` 定义在 EPSG:4326，追加时会生成空几何。导入前先用 `ogrinfo` 确认坐标系，在 `ogr2ogr` 中显式指定 `-s_srs EPSG:3857 -t_srs EPSG:4326` 并加 `-lco GEOMETRY_NAME=geom`，必要时先写入临时表验证 `COUNT(*) FILTER (WHERE geom IS NOT NULL)`，再把 `geom` 复制回正式表（同步填充 `wkb_geometry`），最后删除临时表。
- 沿用这次流程：-s_srs EPSG:3857 -t_srs EPSG:4326，并使用临时表验证几何后再插入正式表。

## 8. 与后端 Tile Catalog 对齐（重要）

后端会根据 bbox 计算相交的 `tile_id`，并在 WFS 查询里附带 `tile_id IN (...)` 来限流/提速。请确保：

1) PostGIS 的 `buildings` 表里有 `tile_id` 字段并已填充。  
2) 后端 tile catalog 已覆盖你导入的瓦片范围：
   - 默认：`ShadowMap/shadow-map-backend/config/buildingTiles.json`
   - 或通过 `.env` 指定：`BUILDING_WFS_TILE_CATALOG_PATH=/abs/path/to/buildingTiles.json`

更新完 catalog 后，重启后端并用 `/api/wfs-buildings/test` 验证连通性与 tile 过滤是否生效。

按以上流程即可持续扩展新的瓦片数据，保证 WFS/WMS 服务在 `shadowmap` 命名空间下保持统一。
